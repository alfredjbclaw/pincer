# audit-and-plan (Mission)

Frontier-model planner. Run **once per day per repo** at the top of the pincer cycle. Output is a TOML plan that the Goal tier (`triage`) and Control tier (`orchestrator`) read.

This skill is the Mission tier in pincer's five-tier loop. Its pattern is a direct lift of shadcn's `/improve` — frontier model audits, mid-tier models execute.

## What this skill does

1. Reads the repo: source tree, open issues, open PRs, CI status, dependency manifests, changelog, recent commits.
2. Identifies high-leverage work — bugs, debt, missing tests, dependency drift, doc gaps, release blockers.
3. Classifies each item by **leverage** (impact ÷ effort) and **verification path** (how do we know it's done — sandbox test, manual proof, neither).
4. Writes a TOML plan to `~/.openclaw/pincer/plans/<owner-repo>-<YYYY-MM-DD>.toml`.

The plan is the source of truth that `triage` reads next.

## When this skill runs

- Daily cadence (cron: `0 9 * * *` is the default; see config).
- On-demand when the user types `audit` from inside a repo, or `audit owner/repo`.
- Before any release branch cut (the orchestrator will trigger this if the plan is more than 24h old).

## Operating model

This skill spends **frontier model time** (Opus by default). Use it sparingly and make every call count.

### 1. Repo recon

```bash
repo=$(gh repo view --json nameWithOwner --jq .nameWithOwner 2>/dev/null \
       || git remote get-url origin 2>/dev/null | sed -E 's#^git@github.com:##; s#^https://github.com/##; s#\.git$##')
gh repo view "$repo" --json description,defaultBranchRef,visibility,createdAt,updatedAt,stargazerCount
gh issue list --repo "$repo" --state open --limit 100 --json number,title,labels,createdAt,updatedAt,url
gh pr list    --repo "$repo" --state open --limit 100 --json number,title,isDraft,mergeStateStatus,statusCheckRollup,url
gh release list --repo "$repo" --limit 5
gh run list --repo "$repo" --limit 10 --json status,conclusion,name,headBranch
```

### 2. Code surface read

- Read the README, `AGENTS.md` / `CLAUDE.md` if present, `package.json` / `go.mod` / `pyproject.toml` / `Cargo.toml` etc.
- Read the changelog. Note unreleased entries.
- Scan for `TODO` / `FIXME` / `XXX` in source. Tally by file. Do not enumerate — count is sufficient.
- Sample 3–5 hot files (most recently changed, or largest in core dirs) and skim for design risk.

### 3. Verification path classification

Every plan item gets a `verification` field. Pick from:
- **`sandbox`** — runnable by `crabbox run -- <test>` (preferred; most items should be this).
- **`live-proof`** — needs a real account, credential, or external interaction. Cannot be sandbox-verified.
- **`manual-review`** — docs, design notes, judgment calls.
- **`none`** — cannot currently be verified. Flag the verification gap as its own plan item.

### 4. Leverage scoring

Score each item 1–5 on two axes:
- **impact**: user-facing severity, security exposure, release blockingness.
- **effort**: hours of focused work assuming a competent agent runs the loop.

Leverage = `impact / effort`. Sort the plan by leverage descending.

### 5. TOML output

Write the plan to `~/.openclaw/pincer/plans/<owner>-<repo>-<YYYY-MM-DD>.toml`:

```toml
# Plan for <owner>/<repo> — <YYYY-MM-DD>
# Generated by pincer audit-and-plan (Mission tier, Opus).

[meta]
repo            = "owner/repo"
generated_at    = "2026-06-15T09:00:00Z"
mission_model   = "anthropic/claude-opus-4-7"
plan_version    = 1

[summary]
open_issues     = 17
open_prs        = 3
unreleased_changes = "Two bug fixes, one doc update. Releasable as 1.4.1 once #134 lands."
release_gate    = "Blocked by #131 (data-loss bug) and #134 (CI regression)."

# --- Plan items, sorted by leverage descending --------------------------------

[[items]]
id              = "ITEM-001"
kind            = "bug"
title           = "data-loss when concurrent writes race the index flush"
issue_url       = "https://github.com/owner/repo/issues/131"
impact          = 5
effort          = 3
leverage        = 1.67
verification    = "sandbox"
test_command    = "go test ./pkg/index -run TestConcurrentWrites -race"
notes           = "Root cause: index.Flush() not holding the mu. Reporter has repro."

[[items]]
id              = "ITEM-002"
kind            = "ci"
title           = "CI matrix regression on go-1.24"
issue_url       = "https://github.com/owner/repo/issues/134"
impact          = 4
effort          = 2
leverage        = 2.0
verification    = "sandbox"
test_command    = "make test"
notes           = "Likely a dependency pin. Check go.sum diff in #133."

[[items]]
id              = "ITEM-003"
kind            = "docs"
title           = "README missing Apple VZ provider instructions"
impact          = 2
effort          = 1
leverage        = 2.0
verification    = "manual-review"
notes           = "Reuse the install section from the issue thread."
```

### 6. Persistence

Append a one-line audit summary to `~/.openclaw/pincer/log.md`:

```text
2026-06-15T09:00:00Z [mission] audit owner/repo: 17 issues, 3 PRs, 8 plan items, release blocked by ITEM-001 ITEM-002.
```

## Output contract

When invoked from the orchestrator or keeper, this skill must end with:

```text
STATUS: done
FILES: ~/.openclaw/pincer/plans/<owner>-<repo>-<YYYY-MM-DD>.toml
VALIDATION: 8 plan items, sorted by leverage; release gate computed.
NEXT: hand off to triage skill.
```

## Failure modes

- **Repo not found** → STATUS: blocked. NEXT: confirm repo name and access.
- **No issues, no PRs, no recent commits** → STATUS: no_changes. Write an empty-plan TOML with a `[summary]` note.
- **Frontier model unavailable** → STATUS: blocked. NEXT: surface to the orchestrator; do NOT fall back to a cheaper model for the Mission tier. The whole point of this tier is the frontier-model judgment.
