## Workstreams (parallel, isolated worktrees)

### WS-F — Prior-art harvester  (worktree pincer-wt-f / ws-f-priorart)
New `tools/prior_art.py`, opt-in behind a flag, cached per project.
- GitHub code/repo search for a problem statement → rank top-N by **health**
  (stars / recency / has-tests), not raw relevance → fetch key files →
  **distill into a compact `prior-art.md` brief** (structure + patterns + cited
  source URLs), not a raw file dump.
- **Reference, don't transplant.** Never paste license-encumbered code verbatim
  into our public repos; extract approach/structure, cite sources, we generate
  our own impl. Same spirit as the public-GitHub PII rule.
- Feeds both the greenfield planner (`fleet_build` decomposition) and the
  maintainer issue-brief. Config `[prior_art]` knobs, default OFF.
- Tests hermetic (mock GitHub API); no live network in tests.

### WS-G — Terminal escalate / "needs human" state  (pincer-wt-g / ws-g-escalation)
Today `work_history.should_skip` cools an exhausted issue down forever, silently.
- Add an explicit **terminal escalated state**: when attempts exhausted /
  consecutive-failures >= threshold, mark the issue `escalated`, emit a loud
  `AlertThread` "needs human" message, and record it so it is not silently
  retried until a human clears it.
- Touch `tools/work_history.py` (state + helpers) and the orchestrator/loop
  surface that consumes `should_skip`. Hermetic tests.

### WS-H — Cross-loop codex/VM concurrency ceiling  (pincer-wt-h / ws-h-globalgate)
`mem_monitor` guards RAM per-host; nothing caps concurrent codex/VM dispatch
**across** loops. Running multiple projects at once, two loops must not jointly
saturate codex.
- Add a shared, file-backed **global gate**: acquire a global slot (max N across
  all loops on host) before a codex/VM dispatch; release on completion;
  stale-reap crashed holders (reuse `inflight.py` registry patterns). Wire into
  the dispatch path (`runtime_adapter` / loop). Hermetic tests.

## Orchestrator-owned (not delegated)
- **#2 Warm-box toolchain reuse (provision-once)** — GOAL was to stop the
  ~2m40s `apt install` dominating every fix-round. Delivered mechanism: instead
  of rebuilding the crabbox base-image OCI artifact (a separate infra pipeline,
  not verifiable in-session), `sandbox_gate` now supports a reusable ("warm")
  crabbox box (`[sandbox].box_id` / `PINCER_SANDBOX_BOX_ID`). The toolchain is
  provisioned onto it ONCE (host-recorded in `provisioned-boxes.json`), then
  every later run executes the BARE test command on the reused box — no apt
  prelude. The argv contract forbids `||`/`command -v` guards, so the skip
  decision lives host-side. Same outcome (rounds 2..N pay no apt cost), fully
  unit-tested. A literal baked base image remains a future option on top of this.
- **#3 Orphan-reap test on the codex *error/fallback* path** — the #8 wedge was
  codex *erroring* → fallback → dead process lingering on its lock. Explicit test
  that reap fires on the error path, not only on timeout.

## Merge contract (keep merges conflict-free)
- WS-F edits the issue-brief **assembly** function only.
- WS-G edits the **skip/escalation decision** + orchestrator surface only.
- WS-H adds a **new** `global_gate` module + a single dispatch-site call.
- Any residual `loop_spec.py` overlap between F and G is resolved by the
  orchestrator at merge.
